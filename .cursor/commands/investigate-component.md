# Investigate Component

Investigate the component named in the current user request.

Do not modify files.

Trace:

- public entry points;
- callers and consumers;
- implementations;
- Spring wiring;
- configuration properties;
- runtime state;
- tests and custom source sets;
- OTel/JMX/classloader interactions;
- documentation and ADRs;
- Gradle dependencies.

Separate:

- verified facts;
- architectural defects;
- behavioral defects;
- historical debt;
- assumptions;
- insufficient evidence.

Cite exact files, classes, methods, and tests.
